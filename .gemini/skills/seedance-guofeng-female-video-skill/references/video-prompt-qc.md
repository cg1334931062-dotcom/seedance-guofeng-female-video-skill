# Video Prompt QC

Use this file after reading `seeddance-best-practices.md` and before finalizing any Seedance output.

Purpose:

- catch prompt weakness before output
- keep the role stable across角色稿 and视频稿
- make the final prompt easier to debug when the first generation drifts

## Rule 1: 单角色稳定性

### 通过标准

- prompt 里只有一个明确主角色
- 主体身份写得够具体
- 如果画面容易误生多人，明确写出 `only this one character` 的中文自然语言等价约束

### 失败信号

- 提示词读起来像群像海报
- 只有“她”但没有身份和轮廓
- 开场、中段、收尾像三个人都能套进去

### 修正重点

- 重写主体锁定句
- 在一致性约束里补单角色约束

## Rule 2: 角色稿到视频的连续性

### 通过标准

- 视频 prompt 保留角色身份
- 视频 prompt 保留服装配色和结构
- 视频 prompt 保留武器形制或关键配饰
- 视频 prompt 保留最重要的脸谱记忆点

### 失败信号

- 视频只剩气质词，没有角色稿的轮廓
- 视频里服装、武器、发型可以随便换

### 修正重点

- 重写主体锁定句和一致性句
- 把角色稿里最不可丢的 3 到 5 个稳定点重复一次

## Rule 3: 场景关系必须具体

### 通过标准

- 能一句话说清楚角色和空间的关系
- 环境不是背景板，而是在压她、托她、藏她，或被她夺回
- 光线 / 天气 / 动势至少有一项参与表达

### 失败信号

- 只有“在月夜里”“在大殿里”这种地点标签
- 去掉场景后 prompt 基本不变

### 修正重点

- 重写角色与场景关系句
- 补环境压迫 / 释放方式

## Rule 4: 视频风格必须独立清晰

### 通过标准

- 美术基底明确
- 成片对象明确
- 画面质地与情绪边界明确
- 读起来像一个具体成片，而不是抽象风格词堆叠
- 能说清为什么这种风格适合这个角色

### 失败信号

- 只有 `高级`、`电影感`、`仙气`
- 只有 `游戏CG风格`、`水墨感`、`动漫感` 但没有继续细分
- 没说明它到底更像 `剧情过场`、`角色 PV`、`海报延展` 还是 `梦境片段`
- 只说了成片感觉，但没有锁住画面更清、更雾、更柔、更压或更克制
- 风格被镜头动作代替掉了

### 修正重点

- 重写视频风格句
- 先明确 `美术基底 + 成片对象 + 画面质地与情绪边界`
- 再把适配理由回写到镜头和动作

## Rule 5: 镜头逻辑不能是模板骨架

### 通过标准

- 入口镜头和这个角色强相关
- 中段推进不是任意 `近景 -> 中景 -> 定格`
- 收尾方式能留下角色或空间记忆点

### 失败信号

- 分镜能原样套在任何角色身上
- 只有一串镜头名，没有导演关系

### 修正重点

- 重写入口镜头
- 重写空间关系或视线设计

## Rule 6: 动作必须具备身体语言和力量质感

### 通过标准

- 有动作母题
- 有身体语言
- 有力量表达
- 动作是在强化角色，或制造有意的反差

### 失败信号

- 只有 `转身`、`拔剑`、`打斗`、`高燃`
- 动作像任何角色都能做

### 修正重点

- 重写动作母题句
- 重写身体语言和力量表达句

## Rule 7: 风格主轴与动作主轴不能互相打架

### 通过标准

- 明确谁是主轴，谁是辅助轴
- 展示片与动作片不要平均混写
- 风格冲突在进入镜头层前已经被解决

### 失败信号

- 同时想要极致留白和高密度高燃
- 高级展示和猛打爆点同权重并列
- `写意水墨` 和 `强特效游戏 CG`
- `克制展示片` 和 `重游戏 CG 宣发包装`

### 修正重点

- 先定主轴
- 先解决风格冲突
- 再弱化辅助轴到可服务主轴的程度

## Rule 8: 总时长、镜头密度与动作密度要可执行

### 通过标准

- 总时长已经明确确认
- 一条 prompt 通常只保留一个入口、一个中段推进、一个收尾记忆点
- 高光动作不超过一个主爆点
- 镜头密度、动作密度、特效密度与确认时长相匹配

### 失败信号

- 没有明确总时长
- 镜头切换太多
- 运镜、动作、特效都在抢戏
- `5-8 秒` 里试图塞进过多镜头或多次大动作切换

### 修正重点

- 先补总时长
- 删掉次要镜头
- 留住最能定义角色的那一个高光动作

## Rule 9: 参考图策略要明确

### 通过标准

- 无参考图时，用文字重复关键稳定点
- 有参考图时，显式锁脸、发型、服装轮廓、配饰、武器、色彩
- 参考图是保角色，不是替代镜头和场景设计

### 失败信号

- 提到参考图，但没说要锁什么
- 没参考图，却把角色稳定性押在模型自己补完上

### 修正重点

- 重写一致性增强句
- 明确“无参考图版本”与“有参考图增强版本”的差异

## Rule 10: 特效必须服务角色与镜头

### 通过标准

- 特效有明确来源
- 特效有明确时机
- 特效强度没有压过角色本体
- 特效没有盖住脸、武器、轮廓和镜头重点

### 失败信号

- 特效只是漂浮 spectacle
- 特效像游戏 CG，和视频风格不兼容
- 观众先看到特效，后面才想起角色是谁

### 修正重点

- 重写特效强度句
- 重写动作母题句或场景关系句
- 重写特效与镜头关系句

## 快速修正表

- 如果角色不稳，先重写 `主体锁定句`
- 如果动作太泛，先重写 `动作母题 + 身体语言句`
- 如果镜头像模板，先重写 `入口镜头 + 空间关系句`
- 如果场景像摆背景，先重写 `角色与场景关系句`
- 如果服装和武器跑偏，先重写 `一致性约束句`
- 如果风格太空，先重写 `视频风格句`
- 如果风格太宽，先补 `美术基底 + 成片对象 + 画面质地与情绪边界`
- 如果特效抢戏，先降低 `特效强度句`
- 如果特效无来源，先改 `动作母题句` 或 `场景关系句`
- 如果特效盖脸，先改 `特效与镜头关系句`

## 最终静默检查

输出前，默默检查这些事：

- 只有一个主角色吗
- 这是不是同一个角色稿延伸出来的视频
- 场景关系是不是具体而非摆设
- 视频风格是不是独立清楚
- 美术基底是不是明确
- 成片对象是不是明确
- 画面质地与情绪边界是不是明确
- 入口、推进、收尾是不是都存在
- 总时长是不是已经明确，且密度和时长相匹配
- 动作是不是有身体语言和力量质感
- 如果使用特效，特效是不是有来源和时机
- 如果使用特效，特效是不是没有压掉角色和镜头重点
- 主轴冲突是不是已经解决
- 这条 prompt 是自然语言镜头设计，而不是关键词堆砌吗
