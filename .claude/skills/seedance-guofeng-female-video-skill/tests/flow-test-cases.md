# 交互流程测试用例

## 测试用例1：完全没想法的用户
**输入**：
```
/seedance-guofeng-female-video-skill 帮我做个古风视频，不知道做啥
```

**预期流程**：
1. 识别创意状态为"完全没想法"
2. 默认工作模式为"仅视频"
3. 从灵感层开始提问，首先询问角色气质
4. 提供3-4个气质选项（清冷孤绝/明媚英气/魅惑危险/温婉柔美等）
5. 用户选择后逐步引导完成所有核心要素
6. 生成六项收敛回显
7. 用户确认后输出最终提示词

## 测试用例2：有模糊感觉的用户
**输入**：
```
/seedance-guofeng-female-video-skill 帮我设计一个古风女侠的视频，5-8秒，有仙气但又很危险的感觉
```

**预期流程**：
1. 识别创意状态为"有模糊感觉"
2. 识别工作模式为"仅视频"
3. 自动提取时长为6-8秒，标记时长已确认
4. 从脸谱层开始提问，确认具体身份、年龄感等
5. 后续依次确认场景、风格、镜头、动作等
6. 生成收敛回显时包含已提取的信息
7. 输出时自动应用"仙气"和"危险"的风格特征

## 测试用例3：已有明确方向的用户
**输入**：
```
/seedance-guofeng-female-video-skill 我要做一个10秒的仙侠女主视频，角色是清冷剑仙，白衣胜雪，手持仙剑，在云海中御剑飞行，镜头从远到近推进，最后定格在面部特写，风格是电影质感，冷色调，不要特效
```

**预期流程**：
1. 识别创意状态为"已有明确方向"
2. 提取已有的核心要素：
   - 角色：清冷剑仙，白衣，持仙剑
   - 场景：云海
   - 风格：电影质感，冷色调
   - 时长：10秒
   - 镜头：远→近推进，面部特写
   - 动作：御剑飞行
   - 特效：不要
3. 仅询问缺失的关键信息（如记忆点、情绪基调等）
4. 快速进入收敛回显阶段
5. 输出的提示词包含所有用户提供的细节

## 测试用例4：仅角色稿模式
**输入**：
```
/seedance-guofeng-female-video-skill 帮我生成一个古风公主的角色图，要唐代风格，穿襦裙
```

**预期流程**：
1. 识别工作模式为"仅角色稿"
2. 不需要询问视频相关的信息（时长、镜头、动作等）
3. 仅确认角色和场景相关的要素
4. 输出仅包含SeedDream提示词，没有视频相关内容

## 测试用例5：整链路模式
**输入**：
```
/seedance-guofeng-female-video-skill 我需要先做角色立绘，再做视频，角色是狐妖，妩媚又残忍，在古宅里
```

**预期流程**：
1. 识别工作模式为"整链路"
2. 完整确认所有核心要素
3. 输出时包含：
   - SeedDream角色稿提示词
   - 参考图筛选要点
   - 无参考图版本视频提示词
   - 有参考图增强版本视频提示词
4. 两份提示词保持角色设定完全一致

## 测试用例6：中途修改要素
**交互流程**：
1. 用户初始输入："帮我做个女侠视频，6秒"
2. 完成角色、场景、风格确认
3. 用户说："我想修改角色气质，不要英气的，要魅惑的"
4. 技能自动回退到角色气质选择阶段
5. 用户重新选择后，后续流程正常进行
6. 最终输出的提示词使用新的气质设定

## 测试用例7：模糊回答处理
**交互流程**：
1. 技能问："请问角色的核心气质是？"
2. 用户回答："随便"或"都行"或"你看着办"
3. 技能不将此作为确认，生成更具体的对比选项
4. 用户明确选择后才标记为确认

## 测试用例8：要素冲突检测
**输入**：
```
/seedance-guofeng-female-video-skill 温柔娴静的大家闺秀，在战场上激烈打斗，15秒视频
```

**预期**：
1. 质检阶段检测到"温柔娴静"与"激烈打斗"的冲突
2. 在质检报告中给出警告，提示可能存在风格不匹配
3. 建议用户调整其中一项以获得更好效果

## 测试用例9：特效风险检测
**输入**：
```
/seedance-guofeng-female-video-skill 仙气飘飘的仙女，全身布满强烈的发光特效，8秒视频
```

**预期**：
1. 质检阶段检测到特效强度过高
2. 警告用户强特效可能遮挡角色主体
3. 建议适当降低特效强度或调整特效位置

## 测试用例10：时长与镜头匹配检测
**输入**：
```
/seedance-guofeng-female-video-skill 5秒视频，要有全景→中景→特写→特效4个镜头切换
```

**预期**：
1. 质检阶段检测到5秒视频有4个镜头切换
2. 警告用户镜头切换过多可能导致节奏过快
3. 建议减少到2个镜头以内
