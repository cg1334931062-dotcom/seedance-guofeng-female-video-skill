# Seedance 国风女性视频技能

一个交互式的古风女性视频提示词生成工具，作为视频导演引导用户逐步定义六项核心要素，最终输出符合Seedance规范的高质量提示词。

## 功能特性

### 三种工作模式
- **仅视频模式（默认）**：专注于视频提示词生成，优先保证视频主体稳定性
- **整链路模式**：先输出SeedDream静态角色稿提示词，再输出视频提示词，保证角色一致性
- **仅角色稿模式**：仅输出静态角色设定的SeedDream提示词

### 七层交互引导
1. **灵感层**：发现核心记忆点、气质、动作爆点
2. **脸谱层**：锁定角色身份、年龄感、发型、妆容、服装、武器
3. **场景层**：定义空间、光线、与角色关系
4. **视频风格层**：确定美学方向、色彩、手法
5. **镜头层**：设计入口、推进、收尾、总时长
6. **动作层**：定义动作母题、身体语言、力量表达
7. **特效层**：可选，确定特效来源、强度、时机

### 核心优势
- 🎯 **交互式决策**：所有决策点均通过AskUserQuestion提供可点击选项，避免纯文本输入
- 🎨 **美学对比**：每个选项都提供预览描述，明确不同选择的视觉差异
- 🧠 **智能状态管理**：自动跟踪用户选择，计算创作缺口，优先补充最重要的信息
- ✅ **收敛确认**：输出前显式展示六项核心要素，支持任意阶段回退修改
- 🔍 **自动质检**：内置10+项质量检查规则，自动识别潜在问题并给出优化建议
- 📚 **最佳实践**：内置Seedance和SeedDream官方最佳实践，提示词质量有保障

## 技术架构

采用状态机驱动的分层交互架构：

```
┌─────────────────┐    ┌──────────────────┐    ┌─────────────────┐
│  输入处理层      │ →  │  状态管理层       │ →  │  输出生成层      │
│  (解析/识别/路由)│    │ (状态跟踪/缺口计算)│    │ (问题/回显/提示词)│
└─────────────────┘    └──────────────────┘    └─────────────────┘
          ↓                       ↓                       ↓
    ┌─────────────┐         ┌─────────────┐         ┌─────────────┐
    │ 参考资源库   │         │ 状态持久化   │         │ 自动质检引擎 │
    └─────────────┘         └─────────────┘         └─────────────┘
```

### 模块说明
- **index.ts**：技能主入口，流程调度
- **input-parser.ts**：用户输入解析，创意状态识别，工作模式路由
- **state-manager.ts**：状态管理，创作缺口计算，流程阶段控制
- **reference-loader.ts**：参考资源智能加载，选项生成
- **question-generator.ts**：动态问题生成，AskUserQuestion工具调用
- **convergence-builder.ts**：六项收敛回显生成
- **prompt-builder.ts**：提示词自动组装，支持三种工作模式
- **quality-checker.ts**：自动质量检查，生成优化建议

## 快速开始

### 调用方式
```
/seedance-guofeng-female-video-skill [初始描述]
```

### 使用示例
```
/seedance-guofeng-female-video-skill 帮我设计一个古风女侠的视频，5-8秒，有仙气但又很危险的感觉
```

### 交互流程
1. 技能自动识别用户创意状态（完全没想法/有模糊感觉/已有明确方向）
2. 根据创意状态进入对应流程，逐层引导用户确认核心要素
3. 每一步提供2-5个美学对比选项，用户通过点击选择
4. 所有核心要素确认后，展示六项收敛回显供用户确认
5. 用户确认无误后，生成最终提示词包和质量检查报告

## 自定义扩展

### 添加新的参考选项
编辑对应参考资源文件，每行一个选项，格式：
```
- 选项标签 - 选项描述 [预览描述]
```

### 修改质检规则
编辑`quality-checker.ts`中的检查逻辑，添加新的检查项。

### 调整交互流程
编辑`input-parser.ts`和`state-manager.ts`中的状态流转逻辑。

## 最佳实践

1. **明确核心记忆点**：每个角色最好有1-2个独特的记忆点，提升识别度
2. **场景与角色匹配**：场景设计要服务于角色气质，避免风格冲突
3. **时长与镜头匹配**：5-8秒视频建议不超过2个镜头切换，保证叙事清晰
4. **特效适度使用**：特效作为点缀，避免抢戏遮挡角色主体
5. **多轮迭代优化**：生成后可根据效果回退调整相关参数，逐步优化

## 常见问题

### Q: 可以中途修改已经确认的要素吗？
A: 可以，在任意阶段说"我想修改XX"，技能会自动回退到对应阶段重新确认。

### Q: 对选项都不满意怎么办？
A: 可以直接输入自定义描述，技能会记录并继续后续流程。

### Q: 生成的提示词还需要调整吗？
A: 自动生成的提示词已经符合官方最佳实践，可以直接使用。也可以根据具体需求手动调整细节。
